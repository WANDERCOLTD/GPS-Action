# `scripts/trace.ts`

Materialised traceability matrix per **D038 §6** + **D053**.

## Three modes

```bash
npm run trace SCN-20      # forward + reverse lookup for a scenario
npm run trace BU-comments # same, centred on a Build Unit
npm run trace D052        # same, centred on an ADR
npm run trace components/CommentList.tsx   # same, for a file

npm run trace:check       # CI guard — fails on broken refs / coverage gaps
npm run trace:matrix      # regenerate docs/architecture/traceability-matrix.md
```

## What it reads

- `docs/product/scenarios.md` — `### Scenario N` headings + the
  `<!-- @no-code-yet -->` parked-marker
- `docs/architecture/decision-log.md` — ADR headings (both
  `### D0NN ·` and `# D0NN —` styles)
- All `.ts` / `.tsx` / `.js` files under `app/`, `components/`,
  `server/`, `shared/`, `prisma/` — extracts `@build-unit`,
  `@spec`, and legacy `@scenarios` JSDoc tags

## What it writes

- `docs/architecture/traceability-matrix.md` (generated by
  `--matrix`; committed to git; CI fails if drifted)
- Console output for single-ID lookups
- Non-zero exit code from `--check` on issues

## Check failures

`trace:check` fails CI when:

- A scenario has zero backing code AND no `@no-code-yet` marker
- A code file's `@spec` references an unknown SCN-N or D-NNN
- The committed matrix file is drifted (run `trace:matrix` to fix)

`trace:check` warns (but doesn't fail) on:

- Legacy `@scenarios SCN-N` tags — migrate to
  `@spec product/scenarios.md (SCN-N)` per D038

## Worktree-safe exclusion

`EXCLUDED_PATH_FRAGMENTS` (`node_modules`, `.next`, `dist`, `build`,
`.claude/worktrees`) are matched against paths **relative to
REPO_ROOT**, not against absolute paths. This matters when the script
runs from inside `.claude/worktrees/<name>/`: the worktree itself is
REPO_ROOT, so its files' relative paths don't contain
`.claude/worktrees` and are correctly included. From the primary
repo, a nested `.claude/worktrees/<sub>/...` relative path still
matches the fragment and is excluded — keeping in-flight agent work
out of the matrix. (See `fix/trace-worktree-exclusion-scope`.)

## Conventions referenced

- `docs/process/traceability.md` — the discipline doc
- D038 — parent ADR
- D053 — output format + thresholds
- F06 rule 1, F13, F14 — the lint rules whose tags this consumes

## What it does NOT do

- Auto-fix / auto-tag missing references
- IDE integration / language server
- Watch mode / file watching
- Visual graph rendering (Mermaid / Graphviz)
- Cross-event coverage (analytics events)
- Transitive chain expansion
