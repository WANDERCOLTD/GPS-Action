Coordination Board · UI surface sketches

v0.4 · 2026-05-03 · Companion to bu-coordination-board.md · Pre-prototype, mockups not pixel-final

Three surfaces for the Direction-A kanban after the Inbox surfaces were dropped (per Paul + Leonid, 2026-05-03 PM). Direction B (shared inbox) is rejected: communication lives inside each ticket, with a thin notifications pane for alerts. The Broadcast wizard split out to a separate bu-broadcast stub.

Each surface embeds its SVG (rendered crisp at any zoom) plus numbered callouts. Question references like Q15 link back to the open-questions list in the build brief. JPG renderings of all three surfaces live alongside in this directory for tools that don't accept SVG.

Surfaces 1 · Kanban desktop 2 · Ticket detail 3 · Notifications pane

1Kanban desktop

Per-team board. Group name as the page title. Tickets are cards; the column encodes work-state (Recruitment → Preparation → Implementation → Monitoring, configurable). Multi-assignee with overflow chip. Urgent is the only cross-cutting flag.

  1. Page title = group name only (Writers). "Active board" subtitle dropped. Backlog and Done get their own subtitles when you click through the tabs.
  2. Header strip — group name on the left, members/admins meta + settings glyph + a primary outline + Propose to backlog button on the right.
  3. Tabs row below the header — Active board (current) · Backlog (12) · Done (47). Backlog and Done are admin-curated views.
  4. Columns are configurable per group Q3 — defaults shown. Writers might prefer Draft → Edit → Publish → Distribute.
  5. Card face — minimal — title · multi-assignee avatars (3-4 visible, then +N overflow chip) · urgent flag if applicable · last-updated time. Names appear on hover/tap, not beside the avatar.
  6. Unclaimed = warning-subtle yellow background (per Q2 — zero-assignee tickets get a coloured indicator). Combines cleanly with the urgent left-border + chip.
  7. Only Urgent gets a chip — the column already encodes work-state, so Draft / Sent / In-progress chips are redundant. Future cross-cutting flags (Blocked, cultural marker) can occupy the same slot if added.
  8. Mobile collapse Q12 — single timeline scroll with column-name as a coloured tag pill on each card. Phase 2.

2Ticket detail

Where work happens. Communication lives inside the ticket — no separate inbox metaphor. Description (editable), interleaved comments + notes + system events, Comment / Note compose tabs. Right-side meta: assignees, subscribers, shared-with teams, invite-group, key dates.

  1. Title with edit pencil — click-to-edit inline, save on blur or Enter. Description below behaves the same. Permission: any group member can edit, audit trail captures changes Q3.
  2. Status row — column position chip (IMPLEMENTATION), urgent flag if applicable, multi-assignee avatars + "+ assign" affordance, last-updated time on the right.
  3. Three top-bar actions: Subscribe (opt in/out of notifications) · Share with team (route to another team within the admin-pre-set workflow) · Invite group (pull a wider audience to this ticket).
  4. Activity feed — interleaved: ⚙ system events (assigned, moved between columns) · regular comments (notify subscribers) · ▌internal notes (yellow tint, silent — no notification). Chronological order.
  5. Composer — Comment / Note as tabs. Comment notifies subscribers; Note is silent Q7. Channel-selector lives at ticket level (subscribe), not per-message.
  6. Right meta sidebar — Assignees (with author badge + unassign affordance · "+ Add assignee") · Subscribers (count + manage link) · Shared-with team chips (each with restricted permissions: can self-assign and comment, cannot move ticket) · Invite-group affordance · Key dates.
  7. Multi-group routing — Share-with-team is limited by an admin-pre-set workflow per team Q1. Receiving team's members inherit read + comment + self-assign rights but cannot change the ticket's column position.

3Notifications pane

Header bell-icon dropdown. Thin alerts list — not a full inbox. Lifecycle: New → Acknowledged → dismissed. Click any row to open the ticket (auto-acknowledges).

  1. Anchored to header bell icon Q13 — badge shows unread count. Dropdown panel ~600px wide on desktop; full-screen overlay on mobile.
  2. Lifecycle — New (urgent dot indicator, bold) · Acknowledged (faded, read) · Dismissed (× per row, removed from view). Auto-acknowledged when the user opens the linked ticket.
  3. Triggers — @mention (always) · comment on a subscribed ticket · column move · new internal note (only if subscribed) · new assignment to you · share-with action (target team's notifications fire).
  4. Each row shows — source ticket title (link) · group name + "shared" tag if cross-team · reason glyph (📝 note · ⏵ moved · @ mention · ⊕ assigned · 💬 comment) · timestamp · per-row dismiss × .
  5. Cross-team awareness Q10 — notifications from shared-with tickets render with the team name they're shared from (e.g. Newsletter Q3 · Social Media (shared)). The team-picker badge counts these.
  6. Footer — "Mark all acknowledged" (sweep action) · "View all →" (full-page notifications view if the dropdown gets crowded).
  7. Not an inbox — you don't reply to tickets from here. Click → opens the ticket detail (Surface 2) where you respond.
